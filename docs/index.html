hh automative
Установка Сценарий Конфигурация Аналитика LLM

Selenium automation toolkit

Контролируемые отклики на hh.ru с историей, диагностикой и локальной аналитикой.

Проект автоматизирует поиск вакансий, выбор резюме, подготовку писем и сохранение технических следов запуска. По умолчанию безопасен для проверки: dry-run не отправляет отклики.

Быстрый старт GitHub 
$ python -m hh_automative check-config
OK config profiles loaded

$ python -m hh_automative run \
  --profile data-engineer \
  --limit 10 \
  --dry-run

History: data/hh_automative.sqlite3
Events:  data/hh_automative.duckdb
01

Поиск вакансий

Профили поиска описываются в JSON и позволяют держать запросы воспроизводимыми.

02

Отклики

Браузерный flow открывает вакансии, подбирает резюме и фиксирует результат.

03

Диагностика

При сбоях сохраняются скриншоты, HTML и структурированные события.

04

Панель

Streamlit-панель читает DuckDB и показывает запуски, ошибки и app logs.

Quickstart

Минимальный запуск

Создайте виртуальное окружение, установите проект в editable-режиме и заполните локальный `.env`. Секреты не должны попадать в репозиторий. 

python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"
cp .env.example .env

Commands

Основные команды

Проверить конфиг

python -m hh_automative check-config

Войти в hh.ru

python -m hh_automative login

Dry-run поиска

python -m hh_automative run --profile data-engineer --limit 10 --dry-run

Сводка событий

python -m hh_automative analytics

Dashboard

python -m hh_automative dashboard --db-path data/hh_automative.duckdb

Все проверки

python -m scripts.check

Flow

Как устроен запуск

1

Загрузка настроек

Читаются `.env`, профиль поиска и карта резюме из `config/`.

2

Браузерный сценарий

Selenium открывает hh.ru, собирает вакансии и проходит форму отклика.

3

История и дедупликация

SQLite хранит обработанные вакансии, чтобы не повторять один и тот же flow.

4

События и отчеты

DuckDB принимает события автоматизации и логи приложения для панели.

Configuration

Файлы и переменные

Настройки разделены на локальные секреты, профили поиска, карту резюме и пути к хранилищам. Это позволяет менять поисковые сценарии без правки Python-кода.

.envсекреты, пути, LLM-провайдеры
config/search_profiles.jsonпрофили поиска вакансий
config/resumes.jsonresume codes и matching rules
data/hh_automative.sqlite3история обработанных вакансий
data/hh_automative.duckdbаналитические события и app logs
reports/diagnosticsHTML и screenshots при ошибках

Observability

Аналитика и dashboard

Локальная панель показывает результат запусков, причины ошибок, выбранные вакансии, diagnostics paths и записи app logs.

automation_eventsruns, vacancies, errors
app_logslevel, logger, message
dashboardStreamlit over DuckDB

AI assist

LLM для писем и опросов

Проект формирует строгие JSON-промпты для сопроводительных писем и анкет работодателя. Поддерживаются Gemini, Mistral, OpenRouter и локальный Ollama. 

HH_USE_LLM=true
HH_USE_LLM_COVER_LETTER=true
HH_USE_LLM_QUESTIONNAIRE=true
HH_LLM_PROVIDER=gemini
HH_LLM_FALLBACKS=mistral,openrouter,ollama
